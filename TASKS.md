# 第一版实现任务拆分

## 任务原则

第一版任务按“先跑起来，再变好玩”的顺序拆分。

每个阶段都尽量保持可验证，避免一次性写太多代码后难以判断问题来源。

`TUNING.md` 是调参文档，`src/config.js` 是运行时代码配置。实现时先把 `TUNING.md` 中的首版参数同步到 `src/config.js`，游戏代码只读取 `src/config.js`。

## 阶段 1：项目骨架

- 创建 `index.html`。
- 创建 `styles.css`。
- 创建 `src/config.js`。
- 创建 `src/input.js`。
- 创建 `src/fish.js`。
- 创建 `src/game.js`。
- 在 `src/config.js` 中写入首版运行参数。
- 在页面中加载样式和脚本。
- 确认浏览器能打开页面。

验收标准：

- 页面能正常打开。
- Canvas 能显示在页面中。
- `src/config.js` 能作为游戏代码的唯一配置来源。
- 控制台没有基础加载错误。

## 阶段 2：基础画布与横屏适配

- 创建全屏 Canvas。
- 根据窗口尺寸调整 Canvas 分辨率。
- 优先适配手机横屏。
- 处理设备像素比，避免高 DPR 屏幕明显模糊。
- 处理窗口尺寸变化。
- 建立屏幕坐标到 Canvas 逻辑坐标的映射。
- 竖屏时第一版可以保持可用，或显示简单横屏提示。
- 设置基础海洋背景。

验收标准：

- Canvas 铺满屏幕。
- 横屏显示正常。
- 触摸和鼠标坐标能正确映射到 Canvas 内的位置。
- 改变窗口尺寸后画面不会明显错位。

## 阶段 3：游戏循环与状态

- 使用 `requestAnimationFrame` 实现主循环。
- 计算每帧 `delta time`。
- 建立游戏状态：
  - 开始前。
  - 游戏中。
  - 胜利。
  - 失败。
- 实现最小开始和重新开始流程。
- 实现最小界面元素：
  - 开始按钮。
  - 状态文本。
  - 再玩一次按钮。

验收标准：

- 点击开始后进入游戏中状态。
- 状态文本能反映开始前、游戏中、胜利和失败。
- 胜利或失败后可以重新开始。
- 循环运行稳定。

## 阶段 4：玩家鱼与输入控制

- 创建玩家鱼对象。
- 绘制玩家鱼。
- 实现触摸目标点移动。
- 用鼠标事件兼容电脑调试。
- 松手后逐渐减速或停下。
- 限制玩家鱼不离开屏幕。

验收标准：

- 手机上触摸屏幕时，玩家鱼能朝触摸位置移动。
- 电脑上可以用鼠标模拟操作。
- 玩家鱼不会游出屏幕外。

## 阶段 5：普通鱼对象

- 创建普通鱼对象。
- 普通鱼包含位置、方向、速度、体型和类型。
- 绘制不同大小和颜色的鱼。
- 普通鱼能从一侧游向另一侧。
- 离开屏幕后移除。

验收标准：

- 屏幕中能看到其他鱼游动。
- 鱼能从左右两侧进入并离开。
- 离屏鱼会被清理。

## 阶段 6：鱼生成系统

- 按 `TUNING.md` 设置同屏最大鱼数。
- 按基础生成间隔生成鱼。
- 前 10 秒只生成小鱼。
- 10 秒后按比例生成小鱼和大鱼。
- 避免生成体型接近玩家的鱼。
- 小鱼体型范围为玩家体型的 `0.45` 到 `0.8` 倍。
- 大鱼体型范围为玩家体型的 `1.25` 到 `1.8` 倍。
- 第一版不主动生成玩家体型 `0.8` 到 `1.25` 倍之间的接近体型鱼。

验收标准：

- 开局主要出现可吃的小鱼。
- 10 秒后开始出现大鱼。
- 同屏鱼数量不会过多。
- 鱼的大小差异明显。
- 不会主动生成规则判断模糊的接近体型鱼。

## 阶段 7：碰撞、吃鱼与失败

- 实现玩家鱼与普通鱼的碰撞检测。
- 小于或等于玩家体型 `0.8` 倍的鱼被碰到后消失。
- 大于或等于玩家体型 `1.25` 倍的鱼被碰到后进入失败状态。
- 如果意外出现接近体型鱼，碰撞暂时不产生吃鱼或失败效果。
- 碰撞判定略微宽容。
- 更新吃鱼数量和分数。
- 实现最小 HUD，显示吃鱼数量和用时。

验收标准：

- 玩家能吃掉明显更小的鱼。
- 玩家碰到明显更大的鱼会失败。
- 接近体型鱼不会造成规则不清的结果。
- 游戏中能看到吃鱼数量和用时。
- 失败后游戏停止并显示失败状态。

## 阶段 8：成长与胜利

- 吃鱼后按参数增加玩家体型。
- 玩家体型影响可吃鱼范围。
- 玩家体型影响移动速度，但不明显破坏手感。
- 玩家达到目标体型后进入胜利状态。

验收标准：

- 吃鱼后玩家能明显变大。
- 玩家达到目标体型后胜利。
- 胜利后游戏停止并显示胜利状态。

## 阶段 9：界面完善与反馈

- 完善开始界面。
- 完善游戏中 HUD。
- 完善胜利界面。
- 完善失败界面。
- 完善再玩一次按钮样式和位置。
- 添加基础吃鱼反馈。
- 添加基础成长反馈。

验收标准：

- 玩家能清楚看到当前状态。
- 吃鱼数量和用时显示清楚。
- 胜利和失败有明确反馈。
- 游戏结束后能快速再玩一次。

## 阶段 10：试玩与调参

- 在电脑浏览器中试玩。
- 在手机横屏中试玩。
- 检查触摸手感。
- 检查胜利所需时间。
- 检查大鱼出现时机和比例。
- 检查画面是否过乱。
- 根据试玩结果调整 `TUNING.md` 和配置代码。

验收标准：

- 第一版可以完整从开始玩到胜利或失败。
- 手机横屏体验可接受。
- 目标胜利时间大致落在 60 到 120 秒。
- 开局前 10 秒没有致命大鱼。
- 同屏鱼数量不超过参数上限。
- 核心“大鱼吃小鱼”体验成立。

## 提交建议

建议按阶段或小功能分批提交。

- `chore: scaffold project files`
- `feat: add canvas game loop`
- `feat: add player fish movement`
- `feat: add fish spawning`
- `feat: add collision and growth rules`
- `feat: add win and lose states`
